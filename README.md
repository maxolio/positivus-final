# 📈 Positivus — Лендинг для маркетингового агентства

Адаптивный одностраничный сайт (лендинг) для digital-маркетингового агентства Positivus. Проект создан для портфолио с целью демонстрации навыков чистой семантической верстки, адаптивного дизайна, организации структуры стилей и работы с интерактивными элементами.

---

## 🛠️ Стек технологий и инструменты
* **HTML5** — семантическая разметка (использование тегов `header`, `main`, `section`, `footer` и др.).
* **CSS3 / SCSS** — стилизация интерфейса, построение сеток (Flexbox, CSS Grid) и адаптивность.
* **BEM (БЭМ)** — методология именования классов для чистоты и масштабируемости кода.
* **JavaScript (ES6+)** — интерактивные элементы (мобильное меню, слайдеры, аккордеоны).
* **Git & GitHub** — контроль версий и публикация проекта.
* **Repomix** — инструмент для оптимизации и упаковки контекста проекта.

</summary>

На данном этапе развернута базовая экосистема проекта, настроен контроль версий и подготовлена файловая структура для старта верстки.

* **Настройка репозитория и контроль версий**:

* **Подготовка файловой структуры**:

    * Развернут базовый шаблон `index.html` с подключенными метатегами адаптивности (viewport).

* **Интеграция Repomix**:
    * Настроен конфигурационный файл `repomix.config.json` для игнорирования служебных файлов при отправке кода в LLM-системы, что облегчает совместную разработку.

<summary>
 Этап 2: Профессиональная SCSS-архитектура, шрифты и глобальные утилиты
</summary>

На данном этапе была полностью спроектирована и внедрена модульная система стилей на базе препроцессора SCSS, подключены фирменные шрифты и подготовлены инструменты для адаптивной верстки.

* **Модульная архитектура стилей (Dart Sass)**:
    * Создана структура каталогов на основе папки `styles/base/` и главного файла-диспетчера `styles.scss`.
    * Использована современная механика импорта `@use` и проброса модулей `@forward` через служебный файл-интегратор `_index.scss`.
    * Настроена автоматическая компиляция SCSS в CSS с помощью встроенного инструмента WebStorm File Watcher.

* **Типографика и шрифты**:
    * Локально интегрирован геометрический гротеск **Space Grotesk** (начертания Regular 400 и Medium 500) в формате `.woff2`.
    * Описаны правила `@font-face` в изолированном модуле `_fonts.scss`.

* **Глобальные настройки и сброс стилей**:
    * Подключена продвинутая нормализация стилей (`_normalize.scss`) для обеспечения кроссбраузерности, плавной прокрутки и стабильного поведения адаптивных изображений.
    * Внедрены глобальные CSS-переменные в `:root` (`_variables.scss`), определяющие цветовую палитру Positivus (темный основной, фирменный салатовый акцент, серые оттенки), скругления, тени и параметры контейнера.

* **Адаптивность и миксины**:
    * Написаны миксины для работы с медиазапросами (`_media.scss`) с брейкпоинтами для мобильных устройств (767px), планшетов (1023px) и десктопов (1280px).
    * Реализован миксин плавной отзывчивой типографики `fluid-text` с использованием функции `clamp()`.
    * Добавлены вспомогательные утилиты (`_utils.scss`) для работы с ограничивающим контейнером `.container`, скрытием элементов на мобильных/десктопных устройствах и утилита `.visually-hidden` для обеспечения доступности (accessibility).

<summary>
 Этап 3: Верстка адаптивной шапки (Header) и интерактивного мобильного меню
</summary>

На данном этапе была полностью завершена работа над шапкой сайта (Header) и мобильным оверлеем. Реализована чистая семантическая разметка и адаптивные стили для всех типов устройств.

* **Семантическая разметка (HTML5 & BEM)**:
    * Сверстана шапка сайта с использованием семантического тега `<header>`.
    * Меню навигации обернуто в тег `<nav>` для обеспечения лучшей доступности.
    * Интегрировано нативное мобильное меню на базе тега `<dialog>` с формой закрытия `<form method="dialog">`, работающей без написания кастомного JavaScript.
    * Все классы названы в строгом соответствии с методологией БЭМ.

* **Адаптивные стили и компоненты**:
    * Разработаны независимые SCSS-компоненты для кнопок (`_button.scss`), кнопки открытия меню (`_burger-button.scss`) и кнопки закрытия (`_cross-button.scss`).
    * Настроены плавные hover-эффекты и анимация подчеркивания для ссылок меню.
    * Стилизовано мобильное меню (`_mobile-overlay.scss`), которое плавно открывается на весь экран на мобильных устройствах, а на десктопе полностью скрывается с помощью утилиты `.visible-mobile`.
    * Реализована плавная адаптивность высоты шапки и отступов за счет использования кастомной функции `fluid()`.

</details>

<summary>
 Этап 4: Верстка первого экрана (Hero Section) и мобильного слайдера партнеров
</summary>

На данном этапе полностью завершена верстка первого экрана сайта (Hero Section) и интерактивного блока с логотипами партнеров компании.

* **Разметка первого экрана (Hero Section)**:
    * Сверстан главный экран с использованием заголовка первого уровня `<h1>` для улучшения SEO-показателей.
    * Главная иллюстрация Hero-секции интегрирована с пустым атрибутом `alt=""` как декоративный элемент.
    * Для логотипов партнеров создана семантическая разметка в виде списка `<ul>` с заполнением корректных текстовых описаний в атрибутах `alt` (Amazon, Dribbble, HubSpot и др.).

* **Адаптивные решения и позиционирование (Flexbox & Grid)**:
    * Для мобильных устройств спроектировано изменение порядка отображения элементов: за счет обнуления блочности родителя (`display: contents` у `.hero__body`) и переопределения порядковых номеров (`order`), иллюстрация на экранах смартфонов автоматически встает второй, сразу после заголовка `<h1>`.
    * Разработан полностью нативный мобильный слайдер для логотипов партнеров на базе технологии **CSS Scroll Snap**. Логотипы аккуратно скроллятся по горизонтали пальцем и плавно центрируются при остановке, не требуя сторонних JS-библиотек.

* **Оптимизация SCSS-переменных**:
    * Внедрены динамические адаптивные отступы секций через CSS-переменную `--section-padding-y` с использованием интерполяции Sass: `#{fluid(70, 30)}`.

</details>

<summary>
 Этап 5: Верстка секции услуг (Services) и рекламного баннера (CTA)
</summary>

На данном этапе завершена верстка ключевой секции услуг (Services), разработана гибкая система сеток и сверстан рекламный баннер (Call-to-Action).

* **Модульная сетка и гибкие компоненты**:
    * Разработан масштабируемый SCSS-модуль сетки (`_greed.scss`), использующий цикл `@for` для автоматической генерации колоночных классов (от 2 до 6 колонок).
    * Создан переиспользуемый класс `.puddle-bg` для стилизации контрастных текстовых плашек-подложек.

* **Верстка карточек услуг (Services)**:
    * Сверстаны адаптивные карточки услуг на базе Flexbox. Для мобильных экранов настроено перестроение в сетку CSS Grid для правильного взаимного расположения заголовка, ссылки и картинки.
    * Реализовано три цветовых схемы карточек (стандартная серая, фирменная салатовая и контрастная темная) с наследованием цветов для внутренних элементов.
    * Решена проблема адаптивного ужимания изображений внутри флекс-контейнеров на средних экранах за счет связки `flex-shrink: 1` и ограничения минимальной ширины `min-width: 140px`.

* **Рекламный баннер (CTA)**:
    * Сверстан адаптивный блок призыва к действию (`_banner.scss`) с использованием отрицательных отступов (`margin-block`) для выноса фонового персонажа за пределы сетки на десктопе и его аккуратным скрытием на мобильных экранах.

* **Интеграция Favicon**:
    * Сгенерирован и подключен в `<head>` полный пакет современных адаптивных иконок (Favicon) для всех типов устройств и платформ в формате `.svg`, `.png` и `.ico` с поддержкой манифеста веб-приложения.

<summary>
 Этап 6: Верстка секции кейсов (Case Studies) с адаптивной трансформацией
</summary>

На данном этапе завершена верстка секции кейсов (Case Studies) с использованием продвинутых возможностей Flexbox и медиа-запросов для кардинальной перестройки интерфейса под мобильные устройства.

* **Десктопная раскладка (Grid-like Flexbox)**:
    * Сверстана горизонтальная плашка кейсов с общим темным фоном.
    * Для создания элегантных вертикальных разделительных линий использованы селекторы отрицания `:not(:last-child)` и `:not(:first-child)` в связке с динамическими отступами на базе функции `fluid()`.
    * Использовано свойство `flex: 1` для принудительного выравнивания ширины всех трех колонок независимо от объема текстового контента.

* **Мобильная трансформация (Mobile Carousel)**:
    * При переходе на мобильное разрешение структура плашки полностью трансформируется: общий темный фон отменяется, а каждый кейс оборачивается в независимую карточку со своим скруглением углов и внутренними паддингами.
    * Список перестраивается в горизонтальную скролл-ленту с нативным поведением слайдера через `scroll-snap-type: x mandatory` и жестким запретом на сжатие элементов (`flex-shrink: 0`).

</details>
